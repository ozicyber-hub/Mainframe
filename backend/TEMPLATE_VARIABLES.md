# OziReport — Template Variable Reference

Use `<<placeholder>>` markers in your `.docx` report template. No Jinja2, no docxtpl.
Upload your template via **Repository → Report Templates**, then set it as the default for your organisation.

DOCX exports only use the template selected on the report/export request. There is no bundled fallback template.

---

## How it works

| Use case | Syntax | How |
|---|---|---|
| Scalar value | `<<report_title>>` | Any paragraph — generator finds and replaces the text |
| Loop table | `<<f.title>>` in a table row | Generator detects the `<<f.` prefix, clones that row for each finding, fills it, removes the template row |
| Chart | `<<SEVERITY_CHART>>` | A paragraph containing only this marker is replaced with a severity bar chart image |
| Chart | `<<REMEDIATION_CHART>>` | A paragraph containing only this marker is replaced with a remediation priority pie chart |

## Tips

- **Loop detection** — the generator finds the template row by scanning for `<<f.` / `<<r.` / `<<v.` / `<<s.` in cell text. The row can be anywhere in the table (after a header row or not).
- **Multiple loop tables** — you can have two separate tables using `<<f.` markers (e.g. a summary table and a detail table) and both will be populated.
- **Blank fields** — all placeholders return an empty string `''` if no data is present.
- **Date format** — main dates use `DD Month YYYY` (e.g. `16 April 2026`). Remediation tracker uses `DD/MM/YYYY`.
- **HTML** — all rich-text fields are stripped to plain text before insertion.
- **Word safety** — `<<` markers are plain text so Word never corrupts them. No split-run issues.
